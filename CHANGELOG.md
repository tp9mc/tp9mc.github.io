# CHANGELOG — PropFerma Bot & Mini App

Лог всех изменений, внесённых в проект в ходе разработки с Claude.

---

## 2026-04-29 (продолжение сессии)

### Публикация через sendData — без прокси/туннеля

Проблема: Саша не мог публиковать изменения, т.к. cloudflared-туннель
недоступен из российского LTE. Решение: серый кружок теперь отправляет
данные напрямую в бота через `Telegram.WebApp.sendData()`.

- `_start_tunnel`: кнопка-уведомление сменилась с **InlineKeyboard** на
  **ReplyKeyboard**. Открытие приложения через ReplyKeyboard активирует
  `sendData` в Mini App — данные идут напрямую в бот, минуя туннель.
- `on_web_app_data` (publish): при пустом `imgs:{}` (sendData-путь)
  читается существующий `site_edits.json` и изображения сохраняются —
  кастомные картинки больше не затираются при публикации текстовых правок.
- `index.html`: `tg.sendData()` обёрнут в `try/catch` — если приложение
  открыто не через ReplyKeyboard, ошибка перехватывается и выполняется
  фолбэк-запрос к прокси.
- Удалена команда `/publish` (пользователь отклонил как усложнение UX).

## 2026-04-29

### Надёжное удаление proxy-сообщений при старте
- Proxy-сообщения (🏠 Открыть редактор) сохраняются в
  `/tmp/propferma_pending_del.json` сразу после отправки.
- При старте бота `_flush_pending_del` читает файл, удаляет все
  сохранённые сообщения (уже удалённые игнорируются молча) и стирает файл.
- Раньше если бот умирал в течение 30 сек — daemon-трeд авто-удаления
  гиб вместе с процессом и сообщения оставались навсегда.

### Список редакторов — публикация для нескольких пользователей
- `bot_secrets.py`: `EDITOR_CHAT_IDS = [184667104, 356003432]` (Timofeev_S + sasha_rufous).
- При старте туннеля кнопка `?proxy=URL` рассылается всем из `EDITOR_CHAT_IDS`,
  а не только владельцу. Каждый редактор может публиковать изменения с любого
  устройства.

### Автоперезапуск бота после ночного бэкапа
- `backup.sh` шаг 5: читает PID из `/tmp/propferma_bot.lock`, останавливает
  старый процесс, удаляет лок-файл, запускает `bot.py` заново.
  Лог: `~/claude-workspace/backups/tp9mc/backup.log`.

### Скролл к первому слоту при переключении таба
- При нажатии на «Мебель» / «Освещение» / «Материалы» страница плавно скроллит
  к первой паре элементов нового раздела (`scrollIntoView` с задержкой 50 мс).
- В режиме редактирования скролл не выполняется (`if (!_editOn)`) — таб
  переключается, но страница остаётся на месте.

### Отображение автора публикации в статистике
- Убрано условие скрытия `@editor` — теперь `@username` показывается всегда
  в секции «ИЗМЕНЕНИЯ НА САЙТЕ».
- `bot_secrets.py`: добавлена константа `OWNER_USERNAME = 'Timofeev_S'`.
- `bot.py`: `_handle_publish` использует `OWNER_USERNAME` как фолбэк вместо
  строки `'editor'` (когда `initDataUnsafe.user` пустой — e.g. открытие не
  через кнопку бота).

### Фикс блокировки скролла в режиме редактирования
- Первая попытка (`position: fixed` на body) сломала публикацию изменений —
  откатана.
- Итоговое решение: `addEventListener('touchmove', handler, {passive:false})`
  с `preventDefault()` при входе в edit mode, `removeEventListener` при выходе.
  Модальное окно и оверлей генерации из блокировки исключены.

### Telegram-пользователь в логах публикаций
- `index.html`: при публикации в POST-тело передаётся
  `window.Telegram.WebApp.initDataUnsafe.user`.
- `bot.py`: `_handle_publish` извлекает `username` / `first_name` из payload
  и передаёт в `log_event` вместо `'editor'`.

## 2026-04-28 (предыдущая сессия)

### Публикация изменений редактора для всех пользователей
- Второй клик по кнопке редактирования (тёмная точка) публикует изменения,
  если они есть. Отдельная кнопка публикации убрана.
- Снэпшот состояния EDITS + IMG_STORE берётся при входе в edit mode.
  При выходе вычисляется diff: `text`, `prompt`, `upload`, `gen`.
- `bot.py` `_handle_publish`: data-URL картинки сохраняются как файлы в
  `custom_assets/`, метаданные — в `site_edits.json`, git commit + push.
- `index.html`: загрузка `site_edits.json` при старте страницы — изменения
  транслируются всем пользователям (только новые ключи, localStorage не
  перезаписывается).
- CSS: `.edit-dot.pub-ok` (зелёный) / `.edit-dot.pub-err` (красный) — 2 сек
  обратная связь.

### Детальный лог изменений на сайте
- В `changes[]` записываются: тип (`text`/`prompt`/`upload`/`gen`), EID,
  значение (для текста и промта).
- `bot.py` `build_stats_report`: новая секция «ИЗМЕНЕНИЯ НА САЙТЕ» с
  покрытием каждого изменения, `TYPE_RU` словарь для читаемых названий.

### Прокси-сервер для генерации на фронте
- `bot.py`: HTTP-сервер `_GenHandler` на порту 8765 — POST `/` генерирует
  через HF FLUX.1-schnell, POST `/publish` публикует изменения.
- CORS-заголовки для cross-origin запросов из Telegram Mini App.

### Cloudflared туннель
- При старте бота запускается `cloudflared tunnel --url localhost:8765`.
  URL туннеля парсится из stdout, отправляется владельцу как InlineKeyboard
  с кнопкой `🏠 Открыть конструктор` (URL содержит `?proxy=<url>`).
- Уведомление: `disable_notification=True`, автоудаление через 30 сек.
- `index.html`: `?proxy=` параметр на загрузке записывается в localStorage
  (`gen_proxy_url`) — все последующие генерации и публикации идут через
  туннель.

### Кнопка «⚡ Сгенерировать по промту» в модале редактирования картинки
- В модале img-edit появилась кнопка генерации через прокси.
- Блокирующий оверлей `#gen-overlay` со счётчиком секунд на время генерации.
- После успеха картинка записывается в `IMG_STORE` с `source: 'generated'`.

### Лимит генераций на фронте
- Не более 100 платных HF-генераций в час (localStorage `hf_gen_ts`).
  При превышении — алерт, генерация блокируется.

### Фикс промтов для иконок опций (opt-элементы)
- Ключи `prompts.json` имеют формат `opt-japandi-living-...`, а `imgEid`
  генерировал `opt-j-l-...` → исправлено на полные названия стиля и комнаты.
- Lazy-load `prompts.json` при первом входе в edit mode (`_loadPrompts(cb)`).

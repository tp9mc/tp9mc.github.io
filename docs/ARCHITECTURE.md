# Архитектура

```
┌────────────────────────── GitHub Actions (каждые 3 ч) ──────────────────────────┐
│                                                                                 │
│  tools/market_sim ──▶ shops/            (демо-магазины: HTML, sitemap, JSON)    │
│         │                 ▲                                                     │
│         │                 │ настоящий HTTP (локальный сервер в раннере)         │
│         │            tools/parsing ──▶ data/parsing/   (снапшот + метрики)      │
│         │                 │                                                     │
│         ▼                 ▼                                                     │
│  data/ground_truth   tools/matching ──▶ data/matching/ (мэтчи, индекс, метрики) │
│         │                 │                                                     │
│         └────────▶ tools/analytics ──▶ data/summary, data/shared/ (E2E, алерты) │
│                                                                                 │
│  git commit data/ shops/ → push main → POST /pages/builds → сайт обновлён       │
│  tools/po_bots.send_report → дайджест/алерты в Telegram                         │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────── GitHub Actions (каждые 15 мин) ───────┐
│ tools/po_bots.poll_and_reply:                │
│ getUpdates → ответы PO-ботов в групповом чате│
│ → коммит data/bots/state.json                │
└──────────────────────────────────────────────┘

GitHub Pages (main) отдаёт index.html + assets/* + data/*.json —
дашборд рендерится в браузере из тех же JSON, что пишет пайплайн.
```

## Ключевые решения

**Почему симулятор рынка, а не реальные магазины.** Продукты полностью
рабочие: краулер честно ходит по HTTP, парсит три разных типа источников,
ловит 404/капчи/битые страницы; матчер решает настоящую задачу
сопоставления зашумлённых названий. Симулятор даёт два свойства, которых
нет у реального веба: (1) ground truth — precision/recall считаются точно,
а не экспертной оценкой; (2) законность и стабильность — никакого скрапинга
чужих сайтов из CI. Рынок детерминирован от даты: история воспроизводима,
но живёт — цены дрейфуют днём и внутри дня, ассортимент ротируется по
неделям, источники иногда падают.

**Почему статический дашборд.** GitHub Pages не исполняет код — пайплайн
пишет плоские JSON, браузер рендерит. Ноль инфраструктуры, полная история
в git.

**Матчер.** Кандидаты через блокинг (категория + бренд), скор — взвешенная
сумма: IDF-containment токенов (0.34), триграммы (0.12), бренд (0.16),
близость цены (0.12), цвет (0.10), совпадение числовых артикулов (0.16).
Авто-мэтч при score ≥ 0.74 и отрыве от второго кандидата ≥ 0.04; зона
0.50–0.74 уходит в очередь ручной валидации (симулируемые валидаторы,
70 карточек за прогон, точность 98.5%).

**Telegram-боты без сервера.** Ответы в реальном времени даёт «вахтовый»
воркер (`realtime-bots.yml`): job в GitHub Actions живёт ~5.5 часов и
непрерывно слушает getUpdates (long polling, ответы за секунды), а крон
дважды в час ставит следующую смену в очередь concurrency-группы — она
стартует сразу после завершения текущей. Офсеты передаются между сменами
через actions/cache; chat_id — в data/bots/state.json. Разовый ручной опрос
остаётся в `po-bots.yml` (workflow_dispatch), диагностика — в `bot-test.yml`.

## Данные

| Файл | Содержимое |
|---|---|
| `data/own_catalog.json` | собственный каталог, 900 SKU (фиксированный сид) |
| `data/ground_truth.json` | истинное соответствие оффер → SKU (пишет симулятор) |
| `data/parsing/latest.json` / `history.json` | метрики парсинга: полный срез / компактная история |
| `data/parsing/snapshot.json` | все распарсенные офферы последнего прогона |
| `data/matching/latest.json` / `history.json` | метрики мэтчинга |
| `data/matching/state.json` | очередь валидации + подтверждённые мэтчи |
| `data/matching/price_index.json` | топ отклонений цен для таблицы |
| `data/shared/history.json` / `alerts.json` | E2E метрики и журнал алертов |
| `data/summary.json` | сводка для шапки дашборда и ботов |
| `data/bots/state.json` | chat_id и офсеты Telegram |

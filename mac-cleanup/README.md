# MacScrub 🧹

Портативная утилита очистки macOS (аналог CleanMyMac / OnyX) с приятным
нативным интерфейсом, детальными настройками, очисткой следов использования
за период, отчётами и запуском по расписанию.

**Полностью локальная и офлайн** — никаких обращений в интернет, никаких
внешних сервисов и API. Работает на штатных средствах macOS. Написана с
помощью модели **Fable 5**.

> ⚠️ **Важно про безопасность.** Программа удаляет файлы. По умолчанию она
> работает в режиме **анализа** (ничего не трогает) и отправляет файлы в
> **Корзину**, а не удаляет безвозвратно. Каждый путь проверяется по белому
> списку — стереть системные каталоги, `$HOME`, «Документы» и т.п. она
> физически не может. Начинайте с кнопки **«Анализ»**.

---

## Что умеет

**Мусор и временные файлы**
- Кэш и логи приложений пользователя, отчёты диагностики (крэш-логи)
- Сохранённые состояния приложений, Корзина
- Временные каталоги `/var/folders` (C и T)
- Xcode DerivedData, кэши сборщиков (npm/yarn/pip/gradle/CocoaPods), Homebrew
- Кэш шрифтов, вложения Mail

**Следы использования (приватность)**
- Миниатюры QuickLook, `.DS_Store` и следы Finder
- Списки «Недавние» (Finder и приложения)
- История команд терминала (`.zsh_history`, `.bash_history`, …)
- История браузеров: Safari, Chrome, Edge, Brave, Firefox
- Кэш браузеров

**Системные (нужен `sudo`, только с явным включением)**
- Системный кэш `/Library/Caches`, системные логи `/private/var/log`
- Единый журнал системы (`log erase`), системные крэш-логи

**Очистка за период времени** — `этот день`, `эта неделя`, `этот месяц`
или `всё время`. Для файлов фильтр по времени изменения; для истории
браузеров — удаление записей нужного периода прямо в SQLite-базе.

**Отчёты** — каждый запуск пишет отчёт (`.txt` + `.json`) в
`~/Library/Application Support/MacScrub/reports/`. Отчёты можно просмотреть
и удалить из интерфейса или командой `macscrub report clear`.

**Расписание** — запуск по расписанию через `launchd` (ежедневно/еженедельно,
в заданное время, с выбранными категориями и периодом).

## Установка (портативно, без зависимостей)

Нужен только Mac — всё остальное (`zsh`, `osacompile`, `sips`, `launchctl`,
`sqlite3`) уже есть в macOS. Ничего из интернета не скачивается.

### Способ 1 — в одну команду (проще всего)

Скачивает самодостаточный установщик и собирает приложение на рабочем столе:

```bash
curl -fsSL "https://raw.githubusercontent.com/tp9mc/tp9mc.github.io/claude/mac-cleanup-utility-u77uiy/mac-cleanup/scripts/Install-MacScrub.command" | zsh
```

На рабочем столе появится **`MacScrub.app`** — запускайте двойным щелчком.

### Способ 2 — из исходников

```bash
cd mac-cleanup
./scripts/build.sh
```

Скрипт соберёт **`dist/MacScrub.app`** и положит **ярлык на рабочий стол**.

Первый запуск приложения «из интернета» может блокировать Gatekeeper —
откройте правой кнопкой → **«Открыть»**, либо один раз снимите карантин:

```bash
xattr -dr com.apple.quarantine dist/MacScrub.app
```

### Перенос на другой Mac

MacScrub полностью портативен. Есть два пути:

1. Скопировать всю папку `mac-cleanup` и выполнить `./scripts/build.sh` на
   новом Mac (рекомендуется — пересоберёт `.app` под систему).
2. Скопировать готовый `dist/MacScrub.app` и снять карантин командой выше.

## Интерфейс

Двойной щелчок по `MacScrub.app` открывает меню:

- **Анализ** — показывает, сколько мусора найдено, ничего не удаляя.
- **Очистка** — выбор периода, категорий и способа (Корзина / безвозвратно),
  подтверждение, затем очистка. По окончании можно открыть отчёт.
- **Рекомендации** — локальный анализ (офлайн): что безопасно чистить сразу,
  что с осторожностью, сколько места освободится.
- **Отчёты** — просмотр и удаление отчётов.
- **Расписание** — установка/удаление автозапуска.

## Командная строка

Движок работает и без GUI — это обычный скрипт `bin/macscrub`.

```bash
# Анализ (ничего не удаляется)
macscrub scan

# Анализ + локальные рекомендации (офлайн)
macscrub advise --window this-week

# Список категорий
macscrub categories

# Очистить кэши и Корзину за эту неделю (в Корзину), с подтверждением
macscrub clean --window this-week --categories user-caches,trash --apply

# Очистить следы за месяц безвозвратно, без вопросов
macscrub clean --window this-month --categories all-safe --apply --delete --yes

# Системная очистка (через sudo)
sudo macscrub clean --include-system --window this-month --apply

# Отчёты
macscrub report list
macscrub report show
macscrub report clear
```

Полезные опции: `--window`, `--categories`, `--apply`, `--delete`,
`--include-system`, `--exclude <текст>`, `--json`, `--yes`.
Полная справка — `macscrub help`.

### Постоянные настройки

Скопируйте `config/config.example.env` в
`~/Library/Application Support/MacScrub/config.env` и задайте параметры по
умолчанию (период, категории, способ). Флаги командной строки их перекрывают.

## Расписание

```bash
# Ежедневно в 03:00 чистить следы за текущий день
macscrub schedule install --freq daily --at 03:00 --window this-day \
  --categories user-caches,user-logs,quicklook-thumbnails,trash

# Еженедельно (понедельник) в 04:00 — за неделю
macscrub schedule install --freq weekly --weekday 1 --at 04:00 --window this-week

macscrub schedule status
macscrub schedule uninstall
```

Настраивается и из GUI (пункт «Расписание»).

---

## Рекомендации (локально, офлайн)

Команда `macscrub advise` (и пункт «Рекомендации» в интерфейсе) анализирует
результат сканирования прямо на месте, без сети и внешних сервисов, и
подсказывает: что безопасно чистить сразу (кэш/логи/временное), что делать
с осторожностью (история браузеров и прочие следы), сколько места
освободится и какой способ удаления выбран.

```bash
macscrub advise --window this-week --categories all-safe
```

## Удаление

```bash
./scripts/uninstall.sh
```

Удаляет приложение, ярлык и расписание; по запросу — отчёты и настройки.

## Как это устроено

| Файл | Назначение |
|------|------------|
| `bin/macscrub` | Движок очистки + локальный советник (zsh, без зависимостей) |
| `gui/MacScrub.js` | Нативный интерфейс (JXA) |
| `scripts/Install-MacScrub.command` | Самодостаточный установщик (собирает .app на рабочем столе) |
| `scripts/build.sh` | Сборка `.app` + ярлык на рабочий стол |
| `scripts/uninstall.sh` | Полное удаление |
| `config/config.example.env` | Пример постоянных настроек |

Безопасность движка: белый список разрешённых префиксов + чёрный список
критичных каталогов, отказ на пустые/корневые пути и обход `..`, режим
анализа по умолчанию, удаление в Корзину по умолчанию. Историю терминала
файлы **усекаются**, а не удаляются, чтобы shell продолжал работать. Историю
браузеров трогает только при закрытом браузере.
